  3. Turn 3 · @versioning

    Agent

    Question. Per-turn turn-NN.html snapshots vs. a single mutating design.html?

    Recommended: per-turn snapshots. Three reasons:

    1. History without ceremony. git diff turn-03.html turn-04.html shows exactly what one round resolved. No checkout dance.
    2. Reading any turn is complete. Each file embeds the conversation up to that point. Opening turn-03.html shows design state + conversation through turn 3.
    3. Composes cleanly with chat.md. chat.md is canonical; turn-NN.html is the rendered visual snapshot. Independent files, different jobs.

    Cost. File count grows with turns. A 12-turn design has 12 HTML files. Acceptable — each is small (~10KB), Tailwind via CDN shares fetch cache.

    Alternatives considered.

    • Single mutating HTML: one design.html rewritten in place. Pros: no proliferation. Cons: lose per-turn diff-ability; no "design at turn 3" without git-checkout.
    • HTML + per-turn patches: too much ceremony, defeats single-file archivability.
